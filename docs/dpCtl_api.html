dpCtl Python API¶

Data Parallel Control (dpCtl)

DpCtl provides a lightweight Python wrapper over a subset of DPC++/SYCL’s API. The goal of dpCtl is not (yet) to provide a abstraction for every SYCL function. DpCtl is intended to provide a common runtime to manage specific SYCL resources, such as devices and USM memory, for SYCL-based Python packages and extension modules.

Currently, dpCtl has two main features: a global SYCL queue manager and a USM memory manager.

dump_device_info()¶

Print information about the SYCL device.

get_device_name()¶

Returns the name of the device as a string

get_device_type()¶

Returns the type of the device as a device_type enum

get_driver_version()¶

Returns the OpenCL software driver version as a string in the form: major number.minor number, if this SYCL device is an OpenCL device. Returns a string class with the value “1.2” if this SYCL device is a host device.

get_max_compute_units()¶

Returns the number of parallel compute units available to the device. The minimum value is 1.

get_max_num_sub_groups()¶

Returns the maximum number of sub-groups in a work-group for any kernel executed on the device. The minimum value is 1.

get_max_work_group_size()¶

Returns the maximum number of work-items that are permitted in a work-group executing a kernel on a single compute unit. The minimum value is 1.

get_max_work_item_dims()¶

Returns the maximum dimensions that specify the global and local work-item IDs used by the data parallel execution model. The minimum value is 3 if this SYCL device is not of device type info::device_type::custom.

get_max_work_item_sizes()¶

Returns the maximum number of work-items that are permitted in each dimension of the work-group of the nd_range. The minimum value is (1; 1; 1) for devices that are not of device type info::device_type::custom.

get_vendor_name()¶

Returns the device vendor name as a string

has_int64_base_atomics()¶

Returns true if device has int64_base_atomics else returns false.

has_int64_extended_atomics()¶

Returns true if device has int64_extended_atomics else returns false.

dpctl.create_program_from_source()¶

Creates a Sycl interoperability program from an OpenCL source string.

We use the DPPLProgram_CreateFromOCLSource() C API function to create a Sycl progrma from an OpenCL source program that can contain multiple kernels.

Parameters

  • q (SyclQueue) – The SyclQueue for which the SyclProgram is going to be built.

  • source (unicode) – Source string for an OpenCL program.

  • copts (unicode) – Optional compilation flags that will be used when compiling the program.

Returns

A SyclProgram object wrapping the sycl::program returned by the C API.

Return type

program (SyclProgram)

dpctl.device_context()¶

The SYCL queue defined by the “backend:device type:device id” tuple is set as the currently active queue, i.e., a subsequent call to dpctl.get_current_queue() inside the context returns the queue. The active queue is also returned by the context manager and can be directly used without having to call dpctl.get_current_queue().

If a the request queue is not found an exception is raised.
